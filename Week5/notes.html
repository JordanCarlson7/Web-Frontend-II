Errors, exceptions, and warnings

  • Use Stack Traces to trail your code to find errors
  • Error: will stop Javascript from running or even compiling
  • Warning: code will still run, although possibly incorrectly, (not bad enough to crash

The importance of testing and debugging

  • Strict Mode: labels 'poor coding style' as Errors, increases ability to find errors
  • Add 'use strict'; to the first line of js file to enable, or first line of function to apply to that scope only
  • Use Linting tools to enforece style guides
  • Feature Detection method is to see if something exists first before using it
  • if (window.holoDeck) {
    virtualReality.activate();
    }

Debugging in the browser

  • Alerts, and Console.log()
  • debugger; Place in code anywhere you want a BREAKPOINT to occur, hit play to rerun code in browser

Error objects

  • You can create your own error objects for specific uses of your program
  • Error objects can be thrown and caught

Throwing exceptions

  • function squareRoot(number) {
    'use strict';
    if (number < 0) {
    throw new RangeError('You can't find the square root of negative numbers')
    }
    return Math.sqrt(number);
    };
  • use try blocks to run code and gracefully handle errors
  • function imaginarySquareRoot(number) {
    'use strict';
    try {
    return String(squareRoot(number));
    } catch(error) {
    return squareRoot(-number)+'i';
    }
    }

Tests

  • Use TDD, write tests before you code and then code to pass the tests

Testing frameworks
